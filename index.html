Un laboratorio compacto

En apoyo a la comunidad de smash en Latinoamérica y con el deseo de que sigan creciendo en cuanto a nivel, me he dado a la tarea de traer los adaptadores de entrenamiento. Este adaptador nace en función de que actualmente el modo training de smash no es posible practicar tus combos contra un SDI/DI verdadero, por lo cual el adaptador simula el SDI/DI y puedas labbear mejor tus combos.

¿Como se usa?

Plug and play

El adaptador está pensado para conectarse y funcionar automáticamente al puerto 2 del adaptador de GC, se debe tener claro que se necesitan dos controles de GC, uno para ti que eres el que realizara los combos y otro para usarlo con el adaptador ya que se usara el modo training en smash y el adaptador controlara a la CPU simulando DI.

Modos

Modos de uso

El adaptador cuenta con modos de DI y modos de escape, los cuales de puede ir intercambiando conforme se oprimen el D-pad. si se oprime el boton start en el control que tiene el adaptador los modos se apagaran.

Modo DI (D-Pad izquierda)

  • Random DI a la izquierda / derecha: cambia entre DI a la izquierda y a la derecha en cada frame.
  • True DI - DI en una dirección completamente nueva y aleatoria en cada frame.
  • Set DI - DI en la dirección en la que se sostenía la palanca cuando este modo estaba habilitado.
  • Random SDI - la CPU elegirá una de las 4 direcciones cardinales y la SDI de velocidad máxima en esa dirección. Presione inicio para restablecer SDI, y la CPU elegirá una nueva dirección la próxima vez que sea golpeada.
  • Set SDI - la CPU acelerará SDI en la dirección en que se mantuvo el dispositivo cuando se habilitó este modo.

Modo escape (D-Pad Derecho)

  • Airdodge - Airdodges cada dos frames.
  • Saltar - salta cada dos frames.
  • Down B - La CPU usará down especial cada dos frames.

Input Recording and playback

  • Input Recording - Presiona D-Pad abajo para empezar a grabar inputs (hasta 400 frames) presiona D-Pad abajo nuevamente para dejar de grabar inputs.
  • Input Playback - Presiona D-Pad arriba para ejecutar los inputs grabados. Cuando se termine de ejecutar se volveran a ejecutar en un ciclo.
